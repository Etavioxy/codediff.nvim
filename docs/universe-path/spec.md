# Universe Path Syntax

## 格式

```
[VCS:版本@]路径[:定位器][修饰器]
```

## 规则

| 组成 | 语法 | 说明 |
|------|------|------|
| VCS:版本@ | `git:HEAD~N@` `git:分支@` `git:hash@` `svn:rN@` `hg:版本@` | 可选整体；省略为当前文件 |
| 路径 | `%` `相对路径` `/绝对路径` | `%`=当前文件 |
| :定位器 | 见下方详解 | 可选 |
| 修饰器 | `+N^` `+Nv` `-N^` `-Nv` | 可选；紧跟定位器 |
| 正则 | `[字符集]` `{N}` | 在定位器中使用 |
| 转义 | `\特殊字符` | 转义 `|` `/` `[]` `{}` `\` |

---

## 定位器详解

| 语法 | 说明 |
|------|------|
| `:N-M` | 第N到M行 |
| `:/开始|结束/[索引]` | 分隔符匹配 |
| `:/pattern[索引]+{开始|结束}[索引]` | 匹配pattern行 + 向下扩展到块，索引分别可选 |

---

## 修饰器详解

| 语法 | 说明 |
|------|------|
| `+N^` | 向上扩展N行 |
| `+Nv` | 向下扩展N行 |
| `-N^` | 向上收缩N行 |
| `-Nv` | 向下收缩N行 |

---

## 示例

```
%                                → 当前文件
%:45-120                         → 当前文件45-120行
%:/{|}/                          → 当前大括号块
%:/{|}/[2]                       → 第2个大括号块
%:/{|}/[-1]+3^                   → 最后一个块+前3行
git:HEAD~3@%:/{|}/+3^            → git HEAD~3，当前块+前3行
git:main@foo.py:/def|end/[2]     → git main分支，foo.py第2个def块
%:/class Foo[1]+{|}/[1]+5^+3v    → 第1个"class Foo"匹配 + 第1个块 + 前5后3行
git:HEAD~1@%:/['"]{3}|['"]{3}/   → git HEAD~1，Python多行字符串
svn:r100@%:/<!--|-->/            → SVN r100，HTML注释块
%:/\[|\]/                        → 方括号块（转义）
%:/\/*|\*\//                     → C注释块（转义）
%:/if\|fi/[2]                    → 第2个Shell条件块
%:/{|}/-2^                       → 当前块，去掉顶部2行
%:/{|}/+3^+5v                    → 当前块，前扩3后扩5
%:/{|}/-2^-1v                    → 当前块，去掉顶部2行、底部1行
```

## Diff用法

```vim
:DiffText % git:HEAD~3@%          → 当前文件 vs git HEAD~3
:DiffText %:/{|}/+3^ git:HEAD~3@%:/{|}/+3^    → 当前块+前3行 vs git HEAD~3
:DiffText git:HEAD~1@foo.py:/{|}/ git:main@foo.py:/{|}/    → git HEAD~1块 vs git main块
```

## Buffer写回规则

### 流程

```
编辑阶段: buffer（区间内容）← 用户修改
保存触发:
  ├─ 检查原文件是否变动（changedtick/hash）
  ├─ 有变动？ → 报错，不写回
  └─ 无变动？ → 写回 → 重新定位区间
```

### 规则

| 场景 | 行为 |
|------|------|
| 原文件无变动 | 写回成功，更新区间范围（行数变化时调整end_line） |
| 原文件有变动 | 报错拒绝写回，提示用户手动处理 |
| 写回成功后 | 重新定位区间 |
| 多次连续写回 | 每次重新定位，不累积误差 |
